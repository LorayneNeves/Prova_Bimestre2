1- A classe rica pode ser utilizada para definir os metodos de ação, por exemplo adicionar, calcular valor, já a classe anemica serve somente para informar as propriedades e contrutor. A classe rica encapsula o comportamento de adicionar itens ao pedido, além de calcular o valor total. Já a classe anêmica não contém nenhum comportamento, apenas propriedades.

2- O SRP é um dos cinco princípios de design de software que visam facilitar a manutenção, extensibilidade e reutilização do código, tornando-o mais organizado e de fácil compreensão.
O Princípio de Responsabilidade Única (SRP) afirma que uma classe deve ter apenas uma razão para mudar. Isso significa que uma classe deve ter uma única responsabilidade ou tarefa bem definida. Quando uma classe tem várias responsabilidades, ela se torna mais difícil de manter, testar e evoluir. Um exemplo disto é o projeto que fizemos em sala, para manipulação de produtos, o mesmo era utilizado e manipulado somente na API, depois fizemos a melhoria incluindo o Domain, Data e Application

3- Validação de dados é importante para manter o retorno dos dados do lado do cliente de forma correta, por exemplo a validação de CPF, é importante que tenha a validação de quantidade de numeros, se é unico e valido no geral, o mesmo para o e-mail. O motivo de ser importante é para que nao fiquem dados incorretos, duplicados ou inadequados de acordo com a necessidade da sua aplicação.
